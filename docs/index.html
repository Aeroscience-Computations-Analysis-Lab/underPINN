Core Capabilities

Everything you need for PINN research

From simple ODEs to full 3-D turbulence models — one coherent API, no boilerplate.

🧩

Domain Decomposition (FBPINN)

Overlapping subdomains with sigmoid partition-of-unity windows. Attention-augmented gated residual blocks inside each subdomain for complex geometries.

⚡

lax.scan XLA Fusion

Fuse N gradient steps into a single XLA kernel — 50–500× less Python dispatch overhead on GPU. Works with all solvers; callbacks still fire every N epochs.

🎵

FourierMLP

Trainable random Fourier feature embeddings. Essential for high-frequency oscillatory solutions — Helmholtz, wave equations, high-Re flows.

🎯

Residual-Based Adaptivity (RBA)

Element-wise loss weighting that automatically concentrates training on high-residual boundary and collocation points.

🔄

RAR-D Adaptive Resampling

Periodically replaces collocation points proportionally to |residual|^k (Lu et al., 2021). Composable with lax.scan; optionally supply a custom domain sampler.

🔬

Inverse Problems

Joint optimisation of network weights + physics parameters. Recover thermal diffusivity from sparse noisy observations using log-parameterised positivity constraints.

🚀

Transfer Learning

Parameter transfer (different Re / diffusivity) and temporal transfer (extended time horizon). Warm-start any solver from a pre-trained model with one call.

📋

Callbacks + TrainingConfig

ConsoleLogger (via Python logging), EarlyStopping, and a unified TrainingConfig dataclass with runtime validation — no more silent bad inputs.

🌐

3-D Geometry Primitives

Cylinder pipe (interior, wall, inlet, outlet), NACA 4-digit airfoil, arbitrary Shapely polygons, rectangles, intervals — all with boundary-aware sampling.

📊

Benchmark Suite

One command runs all problems across multiple epoch budgets, producing accuracy plots, convergence grids, CSV tables, loss histories, and a Markdown report.

⚙️

YAML-Driven CLI

Every hyperparameter lives in a config file. Cartesian sweep support built-in. No code changes needed to run experiments or sweeps.

💾

Prediction Persistence

Automatic compressed .npz saving at collocation points after every training run. Includes coordinates, PINN outputs, and exact solutions where available.

Framework Design

Clean abstractions, zero boilerplate

Three abstract base classes form the backbone. Existing classes conform via inheritance — no rewrites required.

📁 Package Layout

  • core/ — BasePDE, BaseLoss, BaseSolver, TrainingConfig
  • nn/ — MLP, FourierMLP, FBPINN, Attention
  • pde/ — Burgers, Wave, Helmholtz, Heat, N-S 2D/3D, k-ε, ODE
  • solver/ — FBPINNSolver, SteadySolver, ODESolver, RANSSolver
  • losses/ — PINNLoss (RBA), ODELoss, SteadyLoss
  • callbacks/ — ConsoleLogger, EarlyStopping
  • geometry/ — Pipe, Airfoil, Rectangle, Shapely polygon
  • training/ — RAR-D adaptive resampling
  • runner/ — Dispatch registry, 6 YAML runners
  • utils/ — save_predictions, safe_choice, set_seed, metrics
  • benchmark_utils/ — BenchmarkRunner, evaluators, report

🧬 Core Abstractions

underPINN/core/base.py 
class BasePDE(ABC):
    # Every PDE implements residual()
    @abstractmethod
    def residual(self, params, *args): ...

class BaseLoss(ABC):
    # Returns (total_loss, aux_tuple)
    @abstractmethod
    def __call__(self, params, *args, **kwargs): ...

class BaseSolver(ABC):
    # Unified init + train interface
    @abstractmethod
    def init(self, key): ...
    @abstractmethod
    def train(self, *args, **kwargs): ...

Command-Line Interface

YAML-driven experiments, zero code changes

Run, sweep, inspect, and benchmark any problem from the terminal. Each run saves all artefacts to its own output directory.

🚀 Single Run

Point at a config file. That's it.

$ python -m underPINN run configs/burgers.yaml
$ python -m underPINN run configs/wave.yaml
$ python -m underPINN run configs/helmholtz.yaml
$ python -m underPINN run configs/heat_forward.yaml
$ python -m underPINN run configs/ode.yaml
$ python -m underPINN run configs/pipe_flow.yaml

Cartesian product across any config keys.

Benchmark Suite

Systematic accuracy-vs-epoch analysis

One command trains every registered problem across multiple epoch budgets and produces a complete analysis package.

Programmatic usage 
from underPINN.benchmark_utils import BenchmarkRunner, generate_report

runner = BenchmarkRunner(
    problems=["burgers", "wave", "ode_exp", "helmholtz"],
    epoch_budgets=[500, 1000, 2000, 5000],
    seed=0, fast_only=True, verbose=True,
)
results = runner.run(out_dir="outputs/bench")
runner.save_json("outputs/bench/results.json")
generate_report(results, runner, out_dir="outputs/bench")

Three patterns — pick the one that fits your use case.

  layers: [2, 64, 64, 64, 1]

  nu: 0.01

data:
  T: 2.0
  n_collocation: 6000

  n_bc: 200

  epochs: 5000
  lr    : 1.0e-3
  early_stopping_patience: 400

  ic_weight: 100.0
  bc_weight: 10.0
  rba      : true

  dir: outputs/burgers

Python API — Burgers PINN 
import jax, optax
from underPINN.nn.mlp       import MLP
from underPINN.pde.burgers   import BurgersPDE
from underPINN.losses.loss    import PINNLoss
from underPINN.solver.fbpinn  import FBPINNSolver
from underPINN.core.config   import TrainingConfig
from underPINN.callbacks.logging       import ConsoleLogger
from underPINN.callbacks.early_stopping import EarlyStopping
from underPINN.utils.seed    import set_seed

key = set_seed(0)   # seeds Python, NumPy and returns JAX PRNGKey

model  = MLP(layers=[2, 64, 64, 64, 1])
pde    = BurgersPDE(model, nu=0.01)
loss   = PINNLoss(model, pde, ic_weight=100.0, bc_weight=10.0, rba=True)
solver = FBPINNSolver(model, pde, loss=loss)
solver.init(key)

config = TrainingConfig(
    epochs      = 5000,
    lr          = 1e-3,
    lr_schedule = optax.cosine_decay_schedule(1e-3, 5000, alpha=1e-2),
    batch_r     = 2048,
    log_every   = 500,
    callbacks   = [ConsoleLogger(log_every=500), EarlyStopping(patience=400)],
)
solver.train(*data, config=config)
Transfer Learning — warm-start from source model 
# ── Phase 1: train on source domain ──────────────────────
config_src = TrainingConfig(
    epochs=2500, lr=1e-3,
    callbacks=[ConsoleLogger(), EarlyStopping(patience=300)],
)
solver_src.init(jax.random.PRNGKey(0))
solver_src.train(*data_src, config=config_src)

# ── Phase 2: fine-tune on target domain ──────────────────
config_tgt = TrainingConfig(
    epochs=2000,
    lr=3e-4,  # lower LR for fine-tuning
    callbacks=[ConsoleLogger(), EarlyStopping(patience=200)],
)
solver_tgt.load_params(solver_src.params)  # warm start
solver_tgt.train(*data_tgt, config=config_tgt)

# Transfer converges 2–3× faster than training from scratch
GPU Acceleration — lax.scan + RAR-D 
# Fuse 100 gradient steps per XLA kernel + adaptive resampling
config = TrainingConfig(
    epochs=5000,
    lr=1e-3,
    n_scan_steps=100,      # 50 Python calls instead of 5000
    resample_period=5,     # RAR-D every 5 outer steps (=500 epochs)
    resample_k=1.0,        # probability ∝ |residual|^1
    callbacks=[ConsoleLogger(log_every=500)],
)
solver.train(*data, config=config)

Installation

Get underPINN running

Pure Python — install in editable mode from the repo root so all examples and the CLI resolve imports automatically.

CPU (development)

Terminal 
# Clone the repo
git clone https://github.com/Prashantiitk23/PINN
cd PINN

# Install in editable mode (adds underPINN to sys.path)
pip install -e .

# Verify
python -m underPINN list

GPU (CUDA 12)

Terminal 
# Upgrade JAX with CUDA 12 support
pip install -U "jax[cuda12]"

# Install the framework
pip install -e .

# Confirm GPU is visible
python -c "import jax; print(jax.devices())"
# [CudaDevice(id=0), ...]
